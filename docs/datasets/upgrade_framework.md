# MedReason → GPT-5.4 Upgrade Framework

**System Prompt Version: v4.1** (incorporates PRISM v2 critical review + Gemini/Codex feedback)
**Test Results: 5/5 PASS** — see [v4_prompt_test_results.md](v4_prompt_test_results.md)

## Why Upgrade?

MedReason (32K examples) was generated by **GPT-4o** using PrimeKG knowledge graph paths.
GPT-5.4 has significantly better reasoning — we can upgrade the dataset quality without
needing to rebuild the KG pipeline from scratch.

## What MedReason Already Has (We Keep)

- KG-grounded entity extraction and path finding (PrimeKG)
- Self-verification (71.4% retention from quality filtering)
- Structured format: paths → reasoning process → conclusion
- 7 source datasets covering USMLE, board exams, and clinical scenarios

## What GPT-5.4 Upgrades

| Aspect | GPT-4o (Original) | GPT-5.4 (Upgrade) |
|---|---|---|
| Clinical depth | Pattern matching, cue-driven | Mechanistic pathophysiology |
| Differentials | Post-hoc elimination | Genuine comparison with reasoning |
| Precision | Sometimes over-absolute | Calibrated confidence |
| Conciseness | ~500 words (sometimes padded) | ~300-600 words (dense, mechanistic) |
| Format | Markdown headers, verbose | `<think>` tags, compact |
| Error rate | 71.4% passed verification | Target >90% with xhigh reasoning |

## The Upgrade Pipeline

```
┌─────────────────────────────────────────────────────────┐
│ MedReason Example                                       │
│ ┌─────────────┐  ┌──────────────┐  ┌─────────────────┐ │
│ │ Question    │  │ KG Paths     │  │ Original CoT    │ │
│ │ + Options   │  │ (PrimeKG)    │  │ (GPT-4o)        │ │
│ └──────┬──────┘  └──────┬───────┘  └────────┬────────┘ │
└────────┼────────────────┼───────────────────┼──────────┘
         │                │                   │
         ▼                ▼                   ▼
┌─────────────────────────────────────────────────────────┐
│ GPT-5.4 (xhigh reasoning effort)                       │
│                                                         │
│ Input: Question + Options + Ground Truth + Original CoT │
│                                                         │
│ Task: Produce BETTER reasoning that:                    │
│   1. Goes deeper on pathophysiology                     │
│   2. Does genuine differential diagnosis                │
│   3. Fixes any errors in original reasoning             │
│   4. Is more concise (150-250 words)                    │
│   5. Formatted as <reasoning>...</reasoning>            │
└────────────────────────┬────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│ Self-Verification (Optional)                            │
│                                                         │
│ GPT-5.4 re-answers using ONLY the upgraded reasoning.   │
│ If answer ≠ ground truth → DISCARD                      │
│ (Same method MedReason used, but with GPT-5.4)          │
└────────────────────────┬────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────┐
│ Output: Training-Ready JSONL                            │
│                                                         │
│ {"messages": [                                          │
│   {"role": "user", "content": "Question...\nOptions..."│
│   {"role": "assistant", "content":                      │
│     "<think>\n[upgraded reasoning]\n</think>\n[answer]"}│
│ ]}                                                      │
└─────────────────────────────────────────────────────────┘
```

## Three Upgrade Modes

### Mode 1: Full Upgrade (32K → ~25K)
```bash
python3 scripts/upgrade_medreason.py --count 32682 --batch-size 10
```
- Re-reasons ALL 32K examples with GPT-5.4
- Estimated cost: ~$3,600 * (5.4 pricing / 4o pricing) ≈ $5-8K
- Estimated time: ~12-18 hours

### Mode 2: Domain-Focused (Filter → Upgrade)
```bash
python3 scripts/upgrade_medreason.py --count 5000 --domain longevity --verify
python3 scripts/upgrade_medreason.py --count 5000 --domain preventive --verify
python3 scripts/upgrade_medreason.py --count 2000 --domain regenerative --verify
```
- Filters MedReason for domain-specific examples first
- Adds self-verification pass for maximum quality
- Best for our longevity/preventive focus

### Mode 3: Pilot (100 examples)
```bash
python3 scripts/upgrade_medreason.py --count 100 --batch-size 5 --verify
```
- Quick quality check before scaling
- Compare upgraded vs original reasoning quality

## Quality Comparison Framework

For any batch of upgrades, evaluate with:
1. **Codex blind review** (10 samples, harsh rubric) — as done before
2. **Gemini blind review** (same 10 samples, harsher framework)
3. **Self-verification rate**: What % produce correct answers from reasoning alone?
4. **Reasoning density**: chars per example, signal-to-noise ratio
5. **A/B comparison**: Show original + upgraded side-by-side to evaluator

## Key Design Decisions

1. **We keep the ground truth answer** — GPT-5.4 must reason TO the known answer,
   not generate its own. This prevents answer drift. Triple-redundancy in prompt.

2. **We show the original reasoning** — GPT-5.4 can fix errors and deepen,
   rather than reasoning from scratch (which might miss the KG-grounded paths).

3. **We use `<think>` format** — Direct compatibility with Qwen 3.5 training.
   No post-processing needed.

4. **Self-verification is optional but recommended** — Adds ~50% more API calls
   but catches reasoning that sounds good but leads to wrong answers.

5. **Domain-conditional longevity framing** (v4.1) — Longevity/preventive lens is
   ONLY applied when the question involves chronic disease, metabolic/CV health,
   cancer, or aging. Anatomy, microbiology, and basic science questions get
   domain-matched reasoning without forced longevity framing.

6. **GRADE-style evidence assessment** (v4.1) — When citing guidelines or thresholds,
   note evidence quality (High/Moderate/Low). This teaches the student model
   epistemic calibration, not just medical knowledge.

7. **Error correction** (v4.1) — The prompt can identify and fix factual errors in
   GPT-4o reasoning (e.g., caspases → metacaspases in plants) without changing answers.
